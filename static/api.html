API Reference

ChromaFFmpeg is a self-hosted FFmpeg API. Submit media URLs or upload binary files, process them server-side, and receive back a persistent URL.

All endpoints require the header  X-API-Key: your-secret-key

Authentication

Every request must include your API key in the X-API-Key header. The key is set via the API_KEY environment variable on the server.

curl 
curl http://localhost:9000/health \
  -H "X-API-Key: your-secret-key"
StatusMeaning
401Missing or invalid API key
500API_KEY is not configured on the server
POST

/merge

Merge a video file and an audio file. The strategy field controls how a length mismatch is resolved.

FieldTypeDefaultDescription
video_urlstring—URL of the video file
audio_urlstring—URL of the audio file
strategystringspeed_matchtrim_or_slow · speed_match · trim
folderstringnullOptional named folder to save output into
StrategyVideo longer than audioAudio longer than video
trim_or_slowTrim video to audio length (stream copy)Slow video down to fill audio duration
speed_matchSpeed up video to match audioSlow video down to match audio
trimCut at whichever stream ends firstCut at whichever stream ends first
curl — trim_or_slow 
curl -X POST http://localhost:9000/merge \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-secret-key" \
  -d '{
    "video_url": "https://example.com/clip.mp4",
    "audio_url": "https://example.com/track.mp3",
    "strategy": "trim_or_slow"
  }'
curl — save to folder 
curl -X POST http://localhost:9000/merge \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-secret-key" \
  -d '{
    "video_url": "https://example.com/clip.mp4",
    "audio_url": "https://example.com/track.mp3",
    "strategy": "trim_or_slow",
    "folder": "MyProject"
  }'
response 
{
  "url": "http://localhost:9000/files/3f2a1b4c/output.mp4",
  "filename": "output.mp4",
  "job_id": "3f2a1b4c-5d6e-7f8a-9b0c-1d2e3f4a5b6c"
}
When duration cannot be detected for either file, the server falls back to -shortest and adds a "warning" key to the response.

FieldTypeDefaultDescription
media_urlstring—URL of the source image or video
media_typestringimageimage or video
animationstringzoom_inSee animation values below
durationnumber6.0Output duration in seconds
fpsinteger25Output frame rate
resolutionstring1920x1080Output resolution e.g. 1280x720
folderstringnullOptional named folder
Animation values
zoom_in zoom_out pan_left pan_right pan_zoom

/combine

Concatenate two or more video or audio files into a single output. Use reencode: true for mixed-codec sources.

FieldTypeDefaultDescription
typestring—video or audio
urlsstring[]—List of media URLs (minimum 2)
reencodebooleanfalseRe-encode for mixed codecs/resolutions (slower)
folderstringnullOptional named folder
curl — video (stream copy) 
curl -X POST http://localhost:9000/combine \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-secret-key" \
  -d '{
    "type": "video",
    "urls": [
      "https://example.com/clip1.mp4",
      "https://example.com/clip2.mp4",
      "https://example.com/clip3.mp4"

    "reencode": false

  -H "X-API-Key: your-secret-key" \
  -d '{
    "type": "audio",
    "urls": [
      "https://example.com/part1.mp3",
      "https://example.com/part2.mp3"

    "reencode": false

  "url": "http://localhost:9000/files/abc12345/output.mp4",
  "filename": "output.mp4",

/metadata

Inspect any media URL — returns duration, format, resolution, fps, bitrate, and stream counts via ffprobe. No output file is created.

curl 
curl -X POST http://localhost:9000/metadata \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-secret-key" \
  -d '{"url": "https://example.com/video.mp4"}'
response
{
  "duration_seconds": 47.32,
  "duration_formatted": "0:00:47",
  "format": "mov,mp4,m4a,3gp,3g2,mj2",
  "size_bytes": 4823042,
  "video_streams": 1,
  "audio_streams": 1,
  "width": 1920,
  "height": 1080,
  "fps": 25.0,
  "bitrate_kbps": 2048
}

/upload

Upload any binary file (video, audio, image) via multipart form data and receive a persistent URL. Maximum file size: 500 MB. Accepts an optional folder form field — the folder is created automatically if it does not exist.

curl — general storage 
curl -X POST http://localhost:9000/upload \
  -H "X-API-Key: your-secret-key" \
  -F "file=@/path/to/video.mp4"
curl — into a named folder 
curl -X POST http://localhost:9000/upload \
  -H "X-API-Key: your-secret-key" \
  -F "file=@/path/to/photo.jpg" \
  -F "folder=my-project"
response (no folder)
{
  "url": "http://localhost:9000/files/abc12345/video.mp4",
  "filename": "video.mp4",
  "job_id": "abc12345-0000-0000-0000-000000000006",
  "size_bytes": 4823042
}
response (with folder)
{
  "url": "http://localhost:9000/store/my-project/photo.jpg",
  "filename": "photo.jpg",
  "folder": "my-project",
  "size_bytes": 204800
}

Error Handling

All errors return a consistent JSON body.

error shape
{
  "detail": {
    "error": "FFmpeg processing failed",
    "detail": "...raw FFmpeg stderr output..."
  }
}
StatusMeaning
400Bad request — invalid input or unreachable URL
401Missing or invalid X-API-Key header
404File or folder not found
413Upload exceeds 500 MB limit
422Request body validation failed
500FFmpeg or server-side processing error
FFmpeg errorCauseFix
No such file or directoryInput URL unreachableVerify the URL is publicly accessible
Invalid data found when processing inputCorrupted or unsupported containerRe-download or convert the source file
height not divisible by 2Odd pixel dimensionsUse scale=trunc(iw/2)*2:trunc(ih/2)*2
Encoder libx264 not foundFFmpeg binary missing codecUse the full FFmpeg Docker image
Could not determine duration (xfade)Clip has no duration metadataPre-process with /metadata or re-encode
